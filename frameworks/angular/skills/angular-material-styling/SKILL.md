# Goal
Enforce standard Angular Material component usage, reactive forms integration, accessibility (a11y), and SCSS design token theme inheritance across Angular applications.

# Instructions
1. Import Angular Material modules granularly using standalone component imports (e.g., `MatButtonModule`, `MatCardModule`, `MatFormFieldModule`, `MatInputModule`).
2. Inherit all component colors, typography, and elevation from the centralized SCSS theme mixins (`@include mat.all-component-themes($theme)`) or Angular Material 3 (M3) design tokens (`var(--mat-sys-primary)`, `var(--mat-sys-surface)`, etc.).
3. Never hardcode static CSS hex colors (`#3f51b5`) or font-families in component `.scss` files; consume CSS custom properties generated by Angular Material.
4. Structure form fields with explicit accessibility elements: `<mat-label>`, `<mat-hint>`, and `<mat-error>`.
5. Use Material container components (`mat-card`, `mat-toolbar`, `mat-sidenav`, `mat-dialog`) for layout structure, applying Flexbox or CSS Grid for responsive positioning.
6. Handle dark mode and alternate themes by toggling CSS class boundaries on container root elements (e.g., `.dark-theme`).

# Examples
Input: Create a responsive user profile card using Angular Material and SCSS theme.
Output:
```html
<mat-card class="profile-card">
  <mat-card-header>
    <div mat-card-avatar class="profile-avatar"></div>
    <mat-card-title>User Profile</mat-card-title>
    <mat-card-subtitle>Administrator</mat-card-subtitle>
  </mat-card-header>
  <mat-card-content>
    <p>Manage your account settings and preferences.</p>
  </mat-card-content>
  <mat-card-actions align="end">
    <button mat-button color="secondary">Cancel</button>
    <button mat-raised-button color="primary">Save Changes</button>
  </mat-card-actions>
</mat-card>
```

```scss
.profile-card {
  background-color: var(--mat-sys-surface-container);
  color: var(--mat-sys-on-surface);
  border-radius: var(--mat-sys-corner-medium, 12px);
  padding: 1rem;

  .profile-avatar {
    background-color: var(--mat-sys-primary-container);
    color: var(--mat-sys-on-primary-container);
  }
}
```

# Constraints
- Do NOT use `::ng-deep` to override Material component internals unless scoped strictly within a host selector and using Material CSS variables.
- Do NOT use inline `style="..."` attributes for colors, margins, or font sizes.
- Do NOT hardcode arbitrary hex colors (`#ffffff`, `#000000`, etc.) in component `.scss` files.
